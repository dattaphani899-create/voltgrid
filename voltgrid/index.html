How it works

From hardware to dashboard
in three steps

Your charger connects to VoltGrid instantly using the OCPP protocol — no custom firmware required.

01
🔌
Connect your charger
Point your charger's OCPP endpoint to our WebSocket server. Any OCPP 1.6 or 2.0.1 compatible hardware works — Wallbox, ABB, Easee, Alfen and more.
02
📡
Data flows in real time
The charger sends status heartbeats, meter values, and session events. VoltGrid processes and stores everything in your dedicated database.
03
📊
Manage from your dashboard
Monitor every charger live, start or stop sessions remotely, view energy consumption, and track revenue — all from one interface.
04
💳
Bill your customers
Flexible billing by kWh, session duration, or flat rate. Integrated payment processing so you get paid automatically.

Protocol

Built on OCPP —
the open standard

→
// Charger boots up
[2, "abc123", "BootNotification", {
  chargePointModel: "Wallbox Pro",
  vendor: "Wallbox"
}]
←
// Server accepts
[3, "abc123", {
  status: "Accepted",
  currentTime: "2025-04-14T10:00:00Z"
}]
→
// Charging session starts
[2, "def456", "StartTransaction", {
  connectorId: 1,
  idTag: "RFID-7A2C",
  meterStart: 0
}]
→
// Live energy reading
[2, "ghi789", "MeterValues", {
  energyActiveImportRegister: 4820 Wh
}]
  • Industry standard protocol
    OCPP (Open Charge Point Protocol) is the universally adopted standard. Any charger that speaks OCPP can plug into VoltGrid — no vendor lock-in.
  • WebSocket based
    The charger opens a persistent WebSocket connection to our server. Messages flow both ways — the server can send remote commands at any time.
  • OCPP 1.6 and 2.0.1
    We support both versions. OCPP 1.6 is the most widely deployed; 2.0.1 adds smart charging, ISO 15118 Plug & Charge, and enhanced security.
